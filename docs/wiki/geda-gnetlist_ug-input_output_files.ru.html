Входные и выходные файлы

gnetlist берёт один или несколько входных файлов схем и выводит результат их обработки в виде списка соединений.

Файлы схем можно посмотреть и отредактировать с помощью редактора принципиальных схем. Подробности об этом см. в Руководстве пользователя gEDA gschem.

Файлы схем

В файлы схем добавляются компоненты, и каждый из них может быть либо представлением подсхемы, которую нужно включить в проект, либо отдельного прибора или устройства, которое должно в нём использоваться. У компонентов есть выводы, являющиеся представлением либо физических выводов электронных приборов, либо логических связей с электрическими соединениями подсхемы. Подключение компонентов в схеме осуществляется через соединение их выводов посредством соединений или шин. Все указанные элементы могут иметь относящиеся к ним атрибуты, определяющие способ их интерпретации программой gnetlist.

Входных файлов схем можно задавать сколько угодно. Все они будут рассматриваться как одно целое для создания одного списка соединений. Это даёт вам возможность разделить свою схему на несколько листов, каждый из которых будет в отдельном файле, что может быть удобно для очень большого и сложного проекта. Все файлы схем проекта перечисляются в командной строке, например: 

gnetlist power-supply.sch motor-drive.sch

Если у вас есть схема, имя файла которой начинается с дефиса («-»), то после любых других ключей, но перед перечнем входных файлов, необходимо добавить специальный ключ --. Это будет гарантией того, что ни одно из имён указанных вами файлов не будет неправильно интерпретировано и понято как ключ. Например: 

gnetlist -- -tricky-name.sch

Имя выходного файла

По умолчанию gnetlist выводит созданный список соединений в файл под именем «output.net». Для указания другого имени файла можно использовать ключ -o. Например, чтобы результат был выведен в файл «stepper.bom», нужно задать: 

gnetlist -o stepper.bom

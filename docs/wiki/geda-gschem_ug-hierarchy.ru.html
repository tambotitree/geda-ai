Иерархические схемы

Когда в проекте должно использоваться несколько одинаковых подсхем, иногда может быть удобно начертить подсхему отдельно, а затем просто несколько раз добавить её в проект. Таким образом создаётся иерархическая структура схем проекта.

Поиск имеющихся подсхем производится в библиотеке исходных данных. Символы «in-1» и «out-1» применяются в подсхемах для указания того, какие соединения следует сделать доступными для подключения из внешней схемы, то есть той, в которую эти подсхемы входят. Для каждой подсхемы должен быть символ, в котором в атрибутах «source=» указаны страницы схем внутренних цепей подсхемы.

Подробности о том, как настроить пути поиска подсхем для gschem, см. в разделе Настройка gschem.

Чтобы добавить подсхему в схему, нужно, как и обычно, просто вставить представляющий её символ в схему как компонент. В подсхемы, в свою очередь, могут входить и другие подсхемы!

Чтобы увидеть схему внутренних цепей компонента подсхемы, выделите его и в меню выберите Иерархия→Войти в подсхему [Hierarchy→Down Schematic]. Чтобы вернуться обратно в исходную схему после завершения редактирования подсхемы, выберите Иерархия→Наверх [Hierarchy→Up].
